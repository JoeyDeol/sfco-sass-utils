2.1 #mixins.descendant descendant($descendants)

For each valid 'descendant keyword' received via the $descendants argument, mixin wraps the provided @content declarations in a new CSS rule. Note that the mixin validates each 'descendant keyword' against an internally defined whitelist.

Usage

// Input
section {
    @include descendant( first last ) {
        color: blue;
        margin-bottom: 30px;
    }
}

// Output
section > *:first-child,
section > *:last-child, {
    color: blue;
    margin-bottom: 30px;
}
Parameters:
  • $descendants
    Space delimited list of descendant keywords.

2.2 #mixins.dims dims($dims)

Mixin allows for quick setting of the width and height properties on a given element.

If the list argument contains 1x item: both properties will be set to the provided value.

If the list argument contains 2x or more items: width will be set equal to the first value; height will be set equal to the second value

In the case where 2x values are provided, null will be ignored.

NOTE: Mixin is a port of the Bourbon library's 'size', which is currently throwing deprecation warnings.

Usage

// Input
.my-elem {
    @include dims( 100px );
}


// Output
.my-elem {
    width: 100px;
    height: 100px;
}

// Input
.row {
    @include dims( 100% 200px );
}

// Output
.row {
    width: 100%;
    height: 200px;
}

// Input
section {
    @include dims( 100% null );
}

// Output
section {
    width: 100%;
}

// Input
.overlay {
    @include dims( null 100% );
}

// Output
.overlay {
    height: 100%;
}

$dims - A list of values.

2.5 #mixins.on on($states)

For each valid 'state' (eg. pseduo-selector) received via the $states argument, mixin wraps the provided @content declarations in a new CSS rule. Note that the mixin validates each 'state' against an internally defined whitelist.

Usage

// Input
a {
    @include on( hover ) { background: red; }
}

// Output
a:hover {
    background: red;
}

// Input
a {
    @include on( focus active ) { margin-right: 10px; }
}

// Output
a:focus,
a:active {
    margin-right: 10px;
}
Parameters:
  • $states
    The list of 'states' for which to apply the provided styles.

2.6 #mixins.parent parent ($type, $modifier)

Mixin allows for the building of rules based on the parent of the current selector.

By default, including the parent mixin with no arguments will result in a new rule, with * > prepended to the current selector.

If the mixin is called with 1x argument, the '*' will be replaced with the value provided.

If the mixin is called with 2x arguments, the second value will be appended to the first (eg. {TYPE}{MODIFIER}).

Usage

// Input
section {
    @include parent {
        background-color: grey;
    }
}

// Output
* > section {
    background-color: grey;
}

// Input
p {
    @include parent( li ) {
        text-decoration: underline;
    }
}

// Output
li > p {
    text-decoration: underline;
}

// Input
li {
    @include parent( ul, '.is-active' ) {
        font-weight: 700;
    }
}

// Output
ul.is-active li {
    font-weight: 700;
}

$type = * - $modifier = '' -

2.7 #mixins.print-map print-map($map)

Given a map, mixin prints 1x declaration for every key/value pair. NOTE: Mixin DOES NOT support nested maps.

Usage

// Input
p {
    @include print-map( $font-styles-map );
}

// Output
p {
    font-size: 1.4rem;
    font-weight: 500;
    text-transform: uppercase;
}


// NOTE:
// Example above assumes that $font-styles-map includes the following keys: `font-size`; `font-weight`; `text-transform`.

2.8 #mixins.pseudo pseudo($elements)

For each valid pseudo element received via the $elements argument, mixin wraps the provided @content declarations in a new CSS rule.

Note that the mixin validates each 'element' against an internally defined whitelist.

Usage

// Input

    @include pseudo( before ) {
        content: 'This is my before element content';
        display: block;

        background-color: black;
    }
}

// Output
div::before {
    content: 'This is my before element content';
    display: block;
    color: white;
    background-color: black;

2.9 #mixins.set set($props, $value)

Mixin creates 1x declaration for each item in a list of properties ($props). The value of each declaration is set to $value.

Usage

// Input
p {
    @include set( margin padding, 0 );
}

// Output
p {
    margin: 0;
    padding: 0;
}

// Input
img {
    @include set( min-height max-height, 100% );
}

// Output
img {
    min-height: 100%;
    max-height: 100%;
}
